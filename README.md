# Cuisine - 美食知识库项目

> 本地化、可搜索、AI 友好的美食记录知识库

---

## 📚 项目概述

本项目旨在将分散在 WPS 云盘中的美食记录转换为结构化的本地知识库，实现：

- 🔍 **全文搜索**：快速查找餐厅、菜品、区域
- 🤖 **AI 辅助**：让 AI 帮你分析、推荐、总结
- 📊 **版本管理**：Git 追踪每次修改
- 💻 **离线可用**：不依赖云端服务

## 🛠️ 技术栈

### 核心工具

- **Python 3.13.9**: 主脚本语言
- **pandoc 3.8**: Word → Markdown 转换
- **pandas**: Excel 数据处理
- **Git**: 版本控制

### 文件格式

- **Markdown (.md)**: 文字记录
- **CSV (.csv)**: 表格数据
- **JPG/PNG**: 图片文件

## 🔄 更新知识库

### 添加新记录

1. **直接编辑** `.md` 或 `.csv` 文件
2. **从 Word 转换**：`pandoc -f docx -t markdown 新文件.docx > 新文件.md`
3. **从 Excel 转换**：使用 pandas 转为 CSV

### 提交到 Git

```bash
git add 重庆-吃/
git commit -m "添加新的美食记录"
git push
```

### 重新转换（全量）

```bash
python convert_cuisine.py
```

## 💡 使用场景

### 场景 1：找餐厅

```bash
# 找到所有火锅店
grep -rl "火锅" 重庆-吃/

# 查看南山附近的餐厅
cat 重庆-吃/南山/弹子石、上新街、涂山、南山_豆包.csv
```

### 场景 2：价格对比

打开 CSV 文件，用 Excel 筛选：
- `人均 (元)` < 50
- `综合评分` > 4.5

### 场景 3：AI 推荐

```
你是一个美食顾问。根据以下重庆美食知识库的内容，
为我规划一条"大学城美食一日游"路线，要求：
1. 早餐、午餐、晚餐各推荐一家店
2. 人均不超过 100 元
3. 包含不同类型的菜品

[粘贴相关文件内容]
```
