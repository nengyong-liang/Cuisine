# AI 快速参考卡片

> ⚡ 5 秒钟速查：AI 添加内容的核心流程

---

## 🎯 核心原则（必须遵守）

```
先搜索 → 再添加
避免重复 | 遵循结构 | 格式统一
```

---

## 🔍 第一步：搜索（30 秒）

```bash
# 搜索餐厅名
grep -r "餐厅名" 重庆-吃/

# 搜索区域
ls 重庆-吃/ | grep "区域"

# 搜索类型
grep -r "火锅" 重庆-吃/
```

**判断：**
- ✅ 完全相同 → ❌ 不添加，提示已存在
- ✅ 部分相同 → 🔄 合并或补充
- ✅ 完全不同 → ➕ 创建新内容

---

## 📂 第二步：定位（10 秒）

```
三峡广场附近 → 三峡广场沙坪坝站龙湖重庆金沙天街/
南山         → 南山/
杨家坪       → 杨家坪/
牛角沱       → 牛角沱/
石桥铺       → 石桥铺/
磁器口/双碑  → 磁器口、双碑/
重大附近     → 重庆-cqu-吃-食堂、学校附近/
```

---

## 📝 第三步：格式（20 秒）

### Markdown 文件

```markdown
---
source_file: "手动添加_餐厅名.md"
converted_date: "YYYY-MM-DD"
added_by: "AI Assistant"
---

# 餐厅名

## 基本信息
- **类型**: ...
- **位置**: ...
- **人均**: ...
- **评分**: ...

## 推荐菜品
1. ...

## 优点/缺点
- ...
```

### CSV 文件

```python
import pandas as pd
df.to_csv('文件.csv', index=False, encoding='utf-8-sig')
```

### 图片

直接复制到对应目录，在 .md 中引用：
```markdown
![描述](media/图片.jpg)
```

---

## ✅ 第四步：验证（10 秒）

- [ ] 路径正确？
- [ ] 格式正确？
- [ ] 编码正确（CSV: utf-8-sig）？
- [ ] 无重复？
- [ ] Git 提交？

```bash
git add 重庆-吃/
git commit -m "添加：[区域] [餐厅名]"
```

---

## ⚠️ 常见错误

| 错误 | 正确做法 |
|------|---------|
| ❌ 直接添加，不搜索 | ✅ 先 `grep -r` 搜索 |
| ❌ 放在错误目录 | ✅ 按地理位置归类 |
| ❌ CSV 用 utf-8 | ✅ 必须用 `utf-8-sig` |
| ❌ 文件名含特殊字符 | ✅ 只用中文、英文、数字、下划线 |
| ❌ 破坏现有结构 | ✅ 只新增，不删除/重命名 |
